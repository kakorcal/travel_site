• master branch:
  the primary, main, live, production version of product

• should create a new branch for each new feature

• removing from staging area: git reset PATH_TO_FILE

• add to staging and commiting: git commit -am 'MESSAGE'

• create new branch: git branch NEW_BRANCH_NAME

• goto a branch: git checkout BRANCH_NAME

• merge a branch: git merge BRANCH_NAME

• merge branch with github:
  - push individual branch to github: git push origin BRANCH_NAME
  - goto github and click compare & pull request
  - click create pull request
  - click confirm merge
  - delete the branch on github
  - git checkout master
  - git pull origin master
  - delete branch on local: git branch -d BRANCH_NAME

• create and checkout branch: git checkout -b BRANCH_NAME
